## 1. 产品概述和目标用户

### 1.1 产品定位

一款轻量级的本地文件搜索桌面工具，帮助用户通过**文件名**或**文件内容**快速定位电脑中的文件，弥补现有工具（Everything 只搜文件名、Windows 自带搜索太慢）的不足，提供**文件名 + 内容联合搜索**的一体化体验。

### 1.2 目标用户

| 维度 | 描述 |
|------|------|
| **用户类型** | 个人开发者 / 技术爱好者 / 知识工作者 |
| **使用场景** | 在大量本地文件中快速定位目标文件，尤其是「只记得文件内容但忘了文件名」的场景 |
| **技术水平** | 具备基本计算机操作能力，偏好简洁高效的工具 |
| **核心痛点** | ① Everything 不支持内容搜索；② Windows 自带搜索速度慢；③ 需要在多个工具之间切换（文件名用 Everything，内容用 ripgrep） |

### 1.3 产品价值

- **一站式搜索**：文件名搜索 + 内容搜索合二为一，无需切换工具
- **快速响应**：搜索结果秒级返回，不打断工作流
- **轻量无负担**：安装包小，内存占用低，后台静默运行
- **个人定制**：可根据自己的文件目录和使用习惯灵活配置

### 1.4 产品名称

**FileFinder** — 本地文件搜索助手

## 2. 详细功能列表

### 2.1 搜索功能

#### F-01 文件名搜索

| 项目 | 说明 | 状态 |
|------|------|------|
| **功能描述** | 用户输入关键词，在指定目录范围内搜索匹配的文件名 | ✅ 已实现 |
| **匹配模式** | ① 精确匹配：文件名完全等于关键词 ② 模糊匹配：文件名包含关键词（默认） ③ 通配符匹配：支持 `*` 和 `?` 通配符 | ✅ 已实现 |
| **大小写** | 默认不区分大小写，可切换为区分大小写 | ✅ 已实现 |
| **搜索范围** | 默认全盘搜索，支持指定目录 | ✅ 已实现 |
| **实时反馈** | 输入时实时显示搜索结果（防抖 300ms） | ✅ 已实现 |

**文件名搜索技术方案（借鉴 Everything + SQLite 索引）：**

- 首次扫描建立 SQLite 文件索引（`file_index_cache` 表），后续搜索直接查库，无需重复遍历
- 内存缓存层（`SearchCache`）：启动时将索引全表加载到内存，搜索在内存中过滤
- 文件系统监控（`QFileSystemWatcher`）：目录变化时自动同步索引，保持索引实时性
- 评分排序：完全匹配 > 前缀匹配 > 单词边界匹配 > 子串匹配

#### F-02 文件内容搜索

| 项目 | 说明 | 状态 |
|------|------|------|
| **功能描述** | 搜索文件内部的文本内容，定位包含指定关键词的文件 | ✅ 已实现（纯文本） |
| **支持格式** | 见下方「支持的文件格式」章节 | 部分实现 |
| **匹配模式** | ① 普通关键词匹配（默认，已实现） ② 正则表达式匹配（P1） | 🔄 P1 |
| **大小写** | 默认不区分大小写，可切换为区分大小写 | ✅ 已实现 |
| **编码支持** | UTF-8、GBK、GB2312、UTF-16 等常见中文编码自动检测 | ✅ 已实现 |
| **二进制文件** | 自动跳过二进制文件，避免无效搜索 | ✅ 已实现 |

**文件内容搜索技术方案（借鉴 AnyTXT Searcher 设计）：**

AnyTXT Searcher 是当前本地全文检索领域的标杆工具，其核心设计理念值得借鉴：

| AnyTXT 设计要点 | FileFinder 对应方案 |
|------|------|
| **无索引实时扫描**（直接扫描文件内容，省去预处理时间） | P0 阶段采用实时扫描 + 多线程并发搜索；P2 阶段引入全文索引加速 |
| **内存映射（mmap）+ 流式匹配**（边读边搜，不一次性加载全文） | 当前使用 `read_text_file()` 整体读取；P2 优化为 `mmap` 分块流式搜索 |
| **多线程异步索引架构** | 已采用 `ThreadPoolExecutor(max_workers=4)` 并发搜索 |
| **50+ 原生解析器**（PDF/DOCX/XLSX/EPUB/代码等） | P0 实现 `TextParser`（32 种文本/代码格式）；P1 扩展 PDF/Word/Excel 解析器 |
| **增量索引更新**（文件变化时只更新差异部分） | 已通过 `QFileSystemWatcher` 监控 + `_sync_directory()` 实现增量同步 |
| **倒排索引 + 布尔查询**（AND/OR/NOT 组合运算） | P2 阶段引入倒排索引，支持高级查询语法 |
| **索引加密存储**（AES-256） | 暂不实现，保持轻量；敏感数据可选清除 |

## 5. 技术栈

### 5.1 技术选型

| 层级 | 技术选择 | 版本 | 理由 |
|------|----------|------|------|
| **编程语言** | Python | 3.10+ | 开发效率高，生态丰富 |
| **GUI 框架** | PySide6 | ≥ 6.5.0 | Qt 官方 Python 绑定，LGPL 许可证更宽松 |
| **文件遍历** | os.scandir / os.walk | 标准库 | 跨平台文件系统遍历 |
| **文件名搜索** | SQLite 索引 + 内存缓存 | 标准库 | 持久化索引 + 内存快速过滤 |
| **内容搜索** | re（正则引擎）+ 多线程 | 标准库 | Python 内置正则 + ThreadPoolExecutor 并发 |
| **文档解析** | PyMuPDF / python-docx / openpyxl | 第三方库 | 分别解析 PDF / Word / Excel 文本内容 |
| **编码检测** | charset-normalizer | 第三方库 | 自动检测文件编码，解决中文乱码问题 |
| **数据存储** | SQLite 3 | 标准库 | 存储搜索历史、文件索引、用户设置 |
| **打包发布** | PyInstaller | — | 打包为独立 exe，无需用户安装 Python |

### 5.4 性能优化策略

| 策略 | 说明 | 状态 |
|------|------|------|
| **SQLite 持久化索引** | 首次扫描建立索引，后续搜索直接查库，无需重复遍历 | ✅ |
| **内存缓存层** | 启动时将索引全表加载到内存，搜索在内存中过滤 | ✅ |
| **多线程搜索** | 搜索任务在后台线程执行，UI 线程不阻塞 | ✅ |
| **文件系统监控** | 目录变化时自动同步索引，保持索引实时性 | ✅ |
| **批量写入** | 扫描结果 500 条/批写入数据库 | ✅ |
| **SQLite PRAGMA 优化** | WAL 模式 + 16MB 缓存 + 64MB 内存映射 | ✅ |
| **二进制跳过** | 检测文件头魔数快速判断二进制文件并跳过 | ✅ |
| **mmap 流式搜索** | 大文件内存映射 + 边读边搜（借鉴 AnyTXT） | 🔄 P2 |
| **倒排索引** | 内容搜索构建倒排索引，加速全文检索（借鉴 AnyTXT） | 🔄 P2 |
| **分批渲染** | 搜索结果分批加载到 UI，避免一次性渲染大量数据 | 🔄 P1 |
| **增量搜索** | 输入过程中，基于上一次结果进行增量过滤而非重新搜索 | 🔄 P2 |
