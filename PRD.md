## 1. 产品概述和目标用户

### 1.1 产品定位

一款轻量级的本地文件搜索桌面工具，帮助用户通过**文件名**或**文件内容**快速定位电脑中的文件，弥补现有工具（Everything 只搜文件名、Windows 自带搜索太慢）的不足，提供**文件名 + 内容联合搜索**的一体化体验。

### 1.2 目标用户

| 维度 | 描述 |
|------|------|
| **用户类型** | 个人开发者 / 技术爱好者 |
| **使用场景** | 在大量本地文件中快速定位目标文件，尤其是「只记得文件内容但忘了文件名」的场景 |
| **技术水平** | 具备基本计算机操作能力，偏好简洁高效的工具 |
| **核心痛点** | ① Everything 不支持内容搜索；② Windows 自带搜索速度慢；③ 需要在多个工具之间切换（文件名用 Everything，内容用 ripgrep） |

### 1.3 产品价值

- **一站式搜索**：文件名搜索 + 内容搜索合二为一，无需切换工具
- **快速响应**：搜索结果秒级返回，不打断工作流
- **轻量无负担**：安装包小，内存占用低，后台静默运行
- **个人定制**：可根据自己的文件目录和使用习惯灵活配置

### 1.4 产品名称（暂定）

**FileFinder** — 本地文件搜索助手

## 2. 详细功能列表

### 2.1 搜索功能

#### F-01 文件名搜索

| 项目 | 说明 |
|------|------|
| **功能描述** | 用户输入关键词，在指定目录范围内搜索匹配的文件名 |
| **匹配模式** | ① 精确匹配：文件名完全等于关键词 ② 模糊匹配：文件名包含关键词（默认） ③ 通配符匹配：支持 `*` 和 `?` 通配符 |
| **大小写** | 默认不区分大小写，可切换为区分大小写 |
| **搜索范围** | 默认全盘搜索，支持指定目录 |
| **实时反馈** | 输入时实时显示搜索结果（防抖 300ms） |

#### F-02 文件内容搜索

| 项目 | 说明 |
|------|------|
| **功能描述** | 搜索文件内部的文本内容，定位包含指定关键词的文件 |
| **支持格式** | 见下方「支持的文件格式」章节 |
| **匹配模式** | ① 普通关键词匹配（默认） ② 正则表达式匹配（可切换） |
| **大小写** | 默认不区分大小写，可切换为区分大小写 |
| **编码支持** | UTF-8、GBK、GB2312、UTF-16 等常见中文编码自动检测 |
| **二进制文件** | 自动跳过二进制文件，避免无效搜索 |

**支持的文件内容搜索格式：**

| 类别 | 格式 |
|------|------|
| **纯文本** | .txt, .md, .log, .csv, .json, .xml, .yaml, .yml, .ini, .cfg, .conf, .toml, .env, .gitignore |
| **代码文件** | .py, .js, .ts, .html, .css, .java, .c, .cpp, .h, .go, .rs, .rb, .php, .sh, .bat, .ps1, .sql |
| **文档文件** | .pdf（文本层）, .docx, .xlsx, .pptx |
| **其他** | .rtf, .epub（纯文本提取） |

#### F-03 联合搜索

| 项目 | 说明 |
|------|------|
| **功能描述** | 同时按文件名和文件内容条件进行搜索，两个条件为 AND 关系 |
| **交互方式** | 提供两个搜索输入框：「文件名」和「内容」，可分别填写或留空 |
| **逻辑说明** | 两个框都填写时，结果需同时满足两个条件；只填一个时等同于单条件搜索 |

### 2.2 过滤功能

#### F-04 文件类型过滤

| 项目 | 说明 |
|------|------|
| **功能描述** | 按文件扩展名过滤搜索结果 |
| **交互方式** | ① 预设分类按钮：全部 / 文档 / 代码 / 图片 / 视频 / 音频 / 压缩包 ② 自定义扩展名输入框，多个用逗号分隔 |
| **排除功能** | 支持排除指定扩展名（如排除 .dll, .exe） |

#### F-05 高级过滤

| 项目 | 说明 |
|------|------|
| **文件大小** | 指定最小/最大文件大小（如 >1MB, <100MB） |
| **修改日期** | 指定时间范围（今天 / 最近7天 / 最近30天 / 自定义日期范围） |
| **目录范围** | 指定搜索的根目录，支持添加多个目录 |
| **排除目录** | 排除不需要搜索的目录（如 node_modules, .git, __pycache__） |

### 2.3 结果展示

#### F-06 搜索结果列表

| 项目 | 说明 |
|------|------|
| **列表字段** | 文件名、文件路径、文件大小、修改时间、文件类型 |
| **排序方式** | 默认按相关度排序，支持按名称/大小/时间/路径排序 |
| **结果数量** | 显示匹配文件总数，默认最多展示 1000 条，支持加载更多 |
| **路径显示** | 相对路径 + 完整路径 tooltip |

#### F-07 内容预览

| 项目 | 说明 |
|------|------|
| **功能描述** | 选中搜索结果后，在侧边预览面板中显示文件内容 |
| **高亮匹配** | 匹配的关键词在预览中高亮显示 |
| **上下文** | 显示匹配行及其前后各 3 行上下文 |
| **大文件处理** | 超过 5MB 的文件只预览匹配部分，不加载全文 |

#### F-08 文件操作

| 项目 | 说明 |
|------|------|
| **打开文件** | 双击搜索结果，使用系统默认程序打开文件 |
| **打开所在目录** | 右键菜单 → 在资源管理器中打开文件所在目录 |
| **复制路径** | 右键菜单 → 复制文件的完整路径到剪贴板 |

### 2.4 搜索历史

#### F-09 历史记录

| 项目 | 说明 |
|------|------|
| **功能描述** | 自动保存用户的搜索记录，方便快速重复搜索 |
| **存储数量** | 最多保存最近 100 条搜索记录 |
| **交互方式** | 点击搜索框时下拉显示历史记录，支持删除单条或清空全部 |
| **存储位置** | 本地 SQLite 数据库 |

### 2.5 设置

#### F-10 通用设置

| 项目 | 说明 |
|------|------|
| **开机自启** | 可选是否开机自动启动并最小化到系统托盘 |
| **最小化到托盘** | 关闭窗口时最小化到系统托盘而非退出 |
| **全局快捷键** | 设置全局快捷键快速唤起搜索窗口（默认 `Ctrl + Alt + F`） |
| **主题** | 浅色 / 深色 / 跟随系统 |
| **默认搜索目录** | 设置默认的搜索根目录 |
| **排除目录列表** | 管理默认排除的目录列表 |

### 4.2 设计原则

| 原则 | 说明 |
|------|------|
| **简洁优先** | 界面元素精简，避免视觉干扰，核心操作一步到位 |
| **搜索即结果** | 输入关键词后实时显示结果，无需按回车（防抖 300ms） |
| **信息密度适中** | 结果列表展示关键信息（文件名、路径、大小、时间），不拥挤 |
| **键盘友好** | 支持上下箭头选择结果、Enter 打开文件、Esc 关闭窗口 |
| **响应迅速** | 搜索过程中显示加载状态，结果分批渲染避免卡顿 |

### 4.3 交互细节

| 场景 | 交互行为 |
|------|----------|
| **输入搜索** | 实时搜索（防抖 300ms），输入过程中自动过滤结果 |
| **无结果** | 显示友好的空状态提示：「未找到匹配的文件，试试其他关键词」 |
| **搜索中** | 结果区域显示加载动画，状态栏显示「搜索中...」 |
| **大结果集** | 首次加载 100 条，滚动到底部自动加载更多 |
| **选中结果** | 单击选中 → 右侧显示预览；双击 → 打开文件 |
| **快捷键** | `Ctrl+F` 聚焦搜索框；`↑↓` 切换结果；`Enter` 打开；`Esc` 清空/关闭 |
| **窗口大小** | 预览面板可拖拽调整宽度，支持隐藏/显示 |

### 4.4 系统托盘

| 功能 | 说明 |
|------|------|
| **托盘图标** | 显示在系统托盘区，右键弹出菜单 |
| **右键菜单** | 打开主窗口 / 搜索 / 设置 / 退出 |
| **最小化行为** | 关闭窗口时最小化到托盘，不退出程序 |

## 5. 技术栈建议

### 5.1 技术选型

| 层级 | 技术选择 | 版本建议 | 理由 |
|------|----------|----------|------|
| **编程语言** | Python | 3.10+ | 开发效率高，生态丰富，用户熟悉 |
| **GUI 框架** | PyQt6 / PySide6 | 6.5+ | 功能强大的桌面 UI 框架，支持现代界面设计 |
| **文件遍历** | os.walk / pathlib | 标准库 | Python 内置，跨平台文件系统遍历 |
| **文件名搜索** | fnmatch + 自定义索引 | 标准库 + SQLite | 通配符匹配 + 本地索引加速 |
| **内容搜索（纯文本）** | re（正则引擎） | 标准库 | Python 内置正则，支持多编码检测 |
| **内容搜索（文档）** | PyMuPDF + python-docx + openpyxl | 第三方库 | 分别解析 PDF / Word / Excel 文本内容 |
| **编码检测** | chardet / charset-normalizer | 第三方库 | 自动检测文件编码，解决中文乱码问题 |
| **数据存储** | SQLite | 标准库 | 存储搜索历史、文件索引、用户设置 |
| **打包发布** | PyInstaller / Nuitka | — | 打包为独立 exe，无需用户安装 Python |

### 5.4 性能优化策略

| 策略 | 说明 |
|------|------|
| **多线程搜索** | 搜索任务在后台线程执行，UI 线程不阻塞 |
| **文件遍历优化** | 使用 os.scandir() 替代 os.listdir()，性能提升约 2-3 倍 |
| **二进制跳过** | 通过文件头魔数快速判断二进制文件并跳过 |
| **分批渲染** | 搜索结果分批加载到 UI，避免一次性渲染大量数据 |
| **路径缓存** | 缓存已扫描的目录结构，减少重复遍历 |
| **增量搜索** | 输入过程中，基于上一次结果进行增量过滤而非重新搜索 |

## 6. 非功能性需求

### 6.1 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **冷启动时间** | ≤ 3 秒 | 从双击图标到窗口可交互 |
| **文件名搜索响应** | ≤ 1 秒 | 10 万级文件目录下搜索 |
| **内容搜索响应** | ≤ 5 秒 | 在指定目录下搜索纯文本内容（不含大型二进制） |
| **内存占用（空闲）** | ≤ 80 MB | 后台静默时 |
| **内存占用（搜索中）** | ≤ 200 MB | 搜索过程中 |
| **安装包大小** | ≤ 80 MB | 打包后的单文件 exe |
| **CPU 占用（空闲）** | ≤ 1% | 后台静默时 |
| **CPU 占用（搜索中）** | ≤ 50% | 搜索过程中，不影响正常使用 |

### 6.2 安全要求

| 项目 | 说明 |
|------|------|
| **数据本地化** | 所有数据（索引、历史、设置）仅存储在本地，不上传任何信息 |
| **无网络请求** | 程序不发起任何网络连接，完全离线运行 |
| **权限最小化** | 仅请求必要的文件读取权限，不修改用户文件 |
| **用户数据保护** | 搜索历史等隐私数据可选清除，不记录敏感文件信息 |

### 6.3 兼容性要求

| 项目 | 要求 |
|------|------|
| **操作系统** | Windows 10 / 11（64 位）为首要支持；后续可扩展 macOS、Linux |
| **Python 版本** | 3.10 及以上 |
| **屏幕分辨率** | 最低支持 1280×720，推荐 1920×1080 |
| **DPI 缩放** | 支持系统高 DPI 缩放（125%、150%、200%） |

### 6.4 可用性要求

| 项目 | 说明 |
|------|------|
| **错误处理** | 搜索出错时给出友好提示，不弹出异常堆栈 |
| **大文件保护** | 单文件超过 50MB 时跳过内容搜索，避免卡顿 |
| **中断搜索** | 支持中途取消搜索（Esc 键或取消按钮） |
| **进度反馈** | 搜索耗时超过 2 秒时显示进度提示 |
| **配置持久化** | 用户设置（目录、过滤规则、窗口大小等）自动保存，下次启动恢复 |

### 6.5 可维护性要求

| 项目 | 说明 |
|------|------|
| **代码规范** | 遵循 PEP 8 编码规范 |
| **模块化** | 搜索引擎、UI、数据模型分层解耦，便于独立测试和替换 |
| **日志记录** | 关键操作记录日志（搜索、错误、索引更新），日志文件位于 `~/.filefinder/logs/` |
| **配置管理** | 用户配置文件位于 `~/.filefinder/config.json`，支持手动编辑 |
