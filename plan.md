### 阶段 7：文档格式解析 🟠 高优先级

> **借鉴 AnyTXT Searcher 的多格式解析器设计，扩展 ParserRegistry**

| 序号  | 任务                     | 优先级  | 说明                            | 涉及文件                  |
| --- | ---------------------- | ---- | ----------------------------- | --------------------- |
| 7.1 | 实现 PDFParser           | 🟠 高 | 使用 PyMuPDF 提取 PDF 文本内容        | `core/file_parser.py` |
| 7.2 | 实现 DocxParser          | 🟠 高 | 使用 python-docx 提取 Word 文档文本   | `core/file_parser.py` |
| 7.3 | 实现 XlsxParser          | 🟠 中 | 使用 openpyxl 提取 Excel 单元格文本    | `core/file_parser.py` |
| 7.4 | 实现 PptxParser          | 🟢 低 | 使用 python-pptx 提取 PPT 文本      | `core/file_parser.py` |
| 7.5 | 注册新解析器到 ParserRegistry | 🟠 高 | 扩展 `_parsers` 列表              | `core/file_parser.py` |
| 7.6 | 更新 constants.py 扩展名集合  | 🟠 高 | 确保 DOCUMENT\_EXTENSIONS 包含新格式 | `constants.py`        |
| 7.7 | 测试文档格式解析               | 🟠 高 | 准备测试 PDF/Word/Excel 文件        | 手动测试                  |

**实现要点（参考 AnyTXT 设计）：**

- 每个解析器继承 `FileParser(ABC)`，实现 `can_parse()` 和 `parse()` 方法
- `parse()` 方法内部使用 `try-except` 包裹，解析失败返回 `None` 而非抛出异常
- PDF 解析使用 `read_only` 模式，避免加载图片等非文本内容
- Excel 解析使用 `read_only=True` + `iter_rows(values_only=True)` 流式读取
- 大文档解析设置超时和大小限制，避免卡死

## 开发顺序建议

**推荐的开发顺序（按优先级排列）：**

```
阶段 6（Bug 修复）→ 阶段 7（文档解析）→ 阶段 8（预览高亮）
→ 阶段 9（搜索历史）→ 阶段 11（代码重构）→ 阶段 10（高级搜索）
→ 阶段 12（系统集成）→ 阶段 13（性能优化）→ 阶段 14（打包发布）
```

**理由：**

1. **Bug 修复优先**：确保现有功能稳定，避免在不稳定基础上开发新功能
2. **文档解析和预览高亮**：这是用户最需要的核心增强功能
3. **搜索历史**：DAO 层已基本就绪，修复后即可集成
4. **代码重构**：在添加更多功能前重构 main\_window\.py，降低后续开发难度
5. **高级搜索**：正则/过滤等增强功能，优先级低于核心功能
6. **系统集成**：托盘/快捷键等体验优化
7. **性能优化**：借鉴 AnyTXT 的 mmap + 倒排索引，是长期优化方向
