# 本地文件搜索工具需求研究

## 目标
做一个简单易用的本地文件搜索小工具，支持通过文件名或文件内容进行快速搜索，帮助自己高效定位电脑中的文件。

## 调研发现

### 一、现有主流工具分析

#### 1. 文件名搜索类

| 工具 | 平台 | 特点 | 速度 |
|------|------|------|------|
| **Everything** | Windows | 直接读取 NTFS MFT 文件系统表，秒级搜索百万级文件 | ⭐⭐⭐⭐⭐ |
| **fd** | 跨平台 | find 命令的现代化替代，自动忽略 .gitignore，语法简洁 | ⭐⭐⭐⭐ |
| **fzf** | 跨平台 | 模糊搜索神器，交互式筛选，支持管道组合 | ⭐⭐⭐⭐ |
| **FindFiles** | 跨平台 | 基于 Tauri + Vue 3 构建，支持文件类型过滤 | ⭐⭐⭐ |

#### 2. 文件内容搜索类

| 工具 | 平台 | 特点 | 速度 |
|------|------|------|------|
| **ripgrep (rg)** | 跨平台 | Rust 编写，比 grep 快 10~300 倍，自动遵守 .gitignore | ⭐⭐⭐⭐⭐ |
| **AnyTXT Searcher** | Windows | 全文索引 + OCR 图片文字检索，支持 100+ 格式 | ⭐⭐⭐⭐⭐ |
| **DocFetcher** | 跨平台 | 开源桌面搜索应用，支持多种文档格式索引 | ⭐⭐⭐⭐ |
| **The Silver Searcher (ag)** | 跨平台 | 专注于代码搜索，速度较快但已被 ripgrep 超越 | ⭐⭐⭐ |

#### 3. AI 语义搜索类（新兴方向）

| 工具 | 特点 |
|------|------|
| **Vexor** | 基于向量嵌入的语义搜索，支持自然语言查询 |
| **fast-search** | Tauri 2 + Rust + Vue 3，AI 驱动的自然语言文件搜索 |

### 二、关键技术方案对比

#### 文件名搜索技术

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **文件系统索引** | 直接读取文件系统元数据（如 NTFS MFT） | 极快，无需预处理 | 仅限特定文件系统 |
| **预建索引** | 扫描文件系统建立文件名索引数据库 | 跨平台，搜索快 | 需要后台维护索引 |
| **实时遍历** | 每次搜索时遍历目录树 | 实现简单，无需索引 | 大量文件时速度慢 |

#### 文件内容搜索技术

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **全文索引（倒排索引）** | 预先提取文本内容并建立索引 | 搜索速度极快 | 需要预处理和存储索引 |
| **实时正则扫描** | 搜索时逐文件读取并用正则匹配 | 无需预处理，结果实时 | 大量文件时速度慢 |
| **AI 向量嵌入** | 将文件内容转为向量，通过语义相似度搜索 | 支持自然语言模糊查询 | 需要模型推理，资源消耗大 |

### 三、现有工具的不足

- **Everything**：只支持文件名搜索，不支持内容搜索
- **ripgrep**：命令行工具，没有图形界面，对非技术用户不友好
- **AnyTXT Searcher**：仅限 Windows 平台，闭源
- **Windows 自带搜索**：速度慢，内容搜索体验差
- **macOS Spotlight**：内容搜索能力有限，不支持自定义过滤
- 大多数工具**不支持同时按文件名和内容联合搜索**

## 核心需求

1. **文件名搜索**：支持精确匹配、模糊匹配、通配符
2. **文件内容搜索**：支持关键词搜索和正则表达式
3. **联合搜索**：可同时按文件名 + 文件内容条件筛选
4. **文件类型过滤**：按扩展名、文件大小、修改日期等过滤
5. **搜索结果预览**：在结果列表中预览匹配的文件内容上下文
6. **跨平台支持**：支持 Windows / macOS / Linux
7. **轻量高效**：启动快，搜索响应快，资源占用低
8. **简洁易用的图形界面**

## 技术选型建议

### 推荐方案：Tauri 2 + Rust + Vue 3

| 层级 | 技术选择 | 理由 |
|------|----------|------|
| **桌面框架** | Tauri 2 | 打包体积小（~5MB），性能优于 Electron，跨平台 |
| **后端逻辑** | Rust | 高性能文件遍历和正则匹配，可调用 ripgrep 库 |
| **前端界面** | Vue 3 + Vite | 开发效率高，组件生态丰富 |
| **文件名搜索** | 预建索引 + 实时遍历结合 | 首次建立索引，后续增量更新 |
| **内容搜索** | ripgrep 核心库 (grep-regex) | 复用 ripgrep 的高性能正则引擎 |
| **全文索引**（可选进阶） | tantivy | Rust 原生全文搜索引擎，轻量高效 |

### 备选方案

- **Python + PyQt/PySide**：开发最快，但打包体积大、性能一般
- **Electron + Node.js**：生态成熟，但内存占用高
- **纯命令行工具 (Rust/Go)**：极简轻量，但缺少图形界面
